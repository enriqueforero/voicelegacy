## [0.1.0] — 2026-05-16

### Added

- **`config`** — Pydantic v2 configuration models (`PipelineConfig`,
  `ReferenceConfig`, `SynthesisConfig`, `WorkspacePaths`) with strict
  validation and mandatory CPML acceptance.
- **`audio`** — Audio preprocessing utilities: mono resample to 22050 Hz,
  spectral denoise (noisereduce), EBU R128 loudness normalization, silence
  trimming, segment slicing.
- **`quality`** — Reference-segment scoring (SNR + duration) with hard gates
  for phone-codec audio, clipping, and out-of-range durations.
- **`corpus`** — Parser for speakerscribe JSON outputs; extracts target-speaker
  segments and writes them as cleaned WAVs.
- **`synthesis`** — XTTS-v2 wrapper (idiap/coqui-tts fork) with VRAM
  management and explicit CPML TOS handling.
- **`persistence`** — SQLite-backed idempotency cache keyed by content hash
  of (text + reference set + config).
- **`pipeline`** — Two-phase orchestrator: corpus build → synthesis.
- **`cli`** — Typer-based command-line interface.
- **`MIN_USABLE_REFERENCE_SEGMENTS`** (=3) — hard floor enforced by
  `run_synthesis`. Below this, XTTS-v2 zero-shot drifts and the result is
  not suitable for legacy delivery — abort with a clear error so the user
  collects more audio before shipping a bad clone.
- **`py.typed`** marker (PEP 561) — `voicelegacy` is now type-check-aware
  for downstream consumers.
- **Companion notebook** `notebooks/notebook_voicelegacy.ipynb` (29 cells,
  generated by `notebooks/build_notebook.py`) optimized for Google Colab
  Free Tier (T4 GPU).
- **Tests** — 53 tests covering `config`, `audio`, `quality`, `corpus`,
  `persistence`, and `pipeline` (including cache-hit / force-resynthesize /
  reference-floor guard paths via mocked XTTS-v2). Coverage 67.8 % overall;
  100 % on `config`, `persistence`, and `__init__`.

### Fixed (pre-release QA pass)

- **`pipeline.run_reference_phase`** — `evaluate_file` was called with the
  package-wide defaults `MIN_REF_DURATION_S=6.0` and `MAX_REF_DURATION_S=30.0`
  instead of the per-run values from `config.reference.min_segment_duration_s`
  and `max_segment_duration_s`. With defaults (corpus filter 4–15 s,
  quality gate 6–30 s) a segment of e.g. 5 s passed the corpus filter,
  was written to disk, then silently failed the quality gate with a
  misleading "duration too short" reason. Now the gate uses the same
  bounds as the corpus filter, eliminating wasted I/O and the false
  failure signal.

### Known limitations (v0.1.0)

- Fine-tuning is **not supported** — only zero-shot inference. Fine-tuning
  needs hardware Colab Free cannot provide; use RunPod / Lambda / Colab Pro+.
- `synthesis.py` is not unit-tested in CI (16.7 % coverage): exercising it
  requires CUDA + a CPML-accepting download of the XTTS-v2 weights (~2 GB).
  The notebook's smoke-test cell (§7) is the contract that validates this
  module end-to-end in the user's own Colab session.
- `cli.py` is not unit-tested (0 % coverage). The CLI is a thin Typer
  wrapper over the same functions exercised by the pipeline tests; a
  Typer `CliRunner`-based test is planned for v0.1.1.

### Notes for first-time users

- Run [speakerscribe](https://github.com/EnriqueForero/speakerscribe) first
  to produce diarized `.json` files. This project consumes those, it does
  not transcribe.
- Phone-call audio is rejected by default (sample rate gate). Override only
  via `ReferenceConfig(min_snr_db=0, ...)` and only if you understand the
  quality tradeoff — the model clones the codec, not the voice.
- Read the **Coqui Public Model License** at https://coqui.ai/cpml before
  setting `accept_coqui_tos=True`. The library refuses to load otherwise.
